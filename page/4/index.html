django中models函数实现i18n

发表于 2014-02-20 | 更新于 2019-02-14

我们在项目中经常会在django的models中添加一些方法，用来作只读属性，但是这些方法怎么作国际化处理呢？请参考下面代码中的is_published属性。



#### 代码实例：

class Entry(models.Model):
    is_active = models.BooleanField(_("is active"), help_text=_(
        "Tick to make this entry live (see also the publication date)."
        " Note that administrators (like yourself) are allowed to preview "
        "inactive entries whereas the general public aren't."),
        default=False,
        db_index=True,
    )
    pub_date = models.DateTimeField(verbose_name=_("Publication date"),
        help_text=_("For an entry to be published, "
        "it must be active and its publication date must be in the past."),
        default=now(),
        db_index=True,
    )

    def is_published(self):
        """
        Return True if the entry is publicly accessible.
        """
        return self.is_active and self.pub_date <= now()
    is_published.boolean = True
    is_published.short_description = ugettext('is published')  # i18n

ubuntu12.04升级到postgis2.0数据库

发表于 2014-02-19 | 更新于 2019-02-14

PostGIS 使得 PostgreSQL 数据库支持对地理数据的存储。2.0 版本包含大量新特性，如支持栅格数据和栅格/矢量分析，支持Postgres9.1扩展系统，允许在共享的边界上处理对象，支持 3D 和 4D 索引等等。本文主要内容是如何在ubuntu12.04上将Postgis升级到2.0版本。

  1. 停止关联程序

    supervisorctl stop uuboard

  2. 升级数据库

    sudo apt-get install python-software-properties
    sudo apt-add-repository ppa:ubuntugis/ppa
    sudo apt-get update
    sudo apt-get install postgresql-9.1-postgis
    sudo apt-get install postgis

  3. 备份数据

    pg_dump -h localhost -p 5432 -U postgres -Fc -b -v -f "olddb.backup" uuboard

  4. 创建新数据库

    CREATE DATABASE uuboard_new WITH OWNER = postgres ENCODING = 'UTF8' TABLESPACE = pg_default LC_COLLATE = 'zh_CN.UTF-8' LC_CTYPE = 'zh_CN.UTF-8' CONNECTION LIMIT = -1;

  5. 创建扩展

    psql -d uuboard_new -c "CREATE EXTENSION postgis;"
    psql -d uuboard_new -c "CREATE EXTENSION postgis_topology;"
    psql -d uuboard_new -f /usr/share/postgresql/9.1/contrib/postgis-2.0/legacy.sql

  6. 恢复数据

    perl /usr/share/postgresql-9.1-postgis/utils/postgis_restore.pl "olddb.backup" | psql -h localhost -p 5432 -U postgres uuboard_new 2> errors.txt

  7. 清理升级数据

    psql -d uuboard_new -f /usr/share/postgresql/9.1/contrib/postgis-2.0/uninstall_legacy.sql

  8. 重命名数据库

    psql -c "ALTER DATABASE uuboard RENAME TO uuboard_backup"
    psql -c "ALTER DATABASE uuboard_new RENAME TO uuboard"

  9. 查询是否升级成功

    psql -d uuboard -c "SELECT postgis_full_version();"

安装hadoop2.0注意事项

发表于 2013-10-19 | 更新于 2019-02-14
  1. 千万不能添加一条指向自己的hosts记录，ubuntu默认会建立，请一定删除。
  2. 安装统一的内网域名解析服务器dnsmasq辅助管理haddop。
  3. dnsmasq域名服务器配置完成后需要重启（service dnsmasq restart）。
  4. 配置统一的rsa密钥，hadoop各节点之间无密码登录。
  5. 需要在整个集群中使用NTP服务。

具体安装方法请参见hadoop官网

自动更新ubuntu/debian

发表于 2013-02-19 | 更新于 2019-02-14

unattended-upgrades是一个 Debian 软件包，用以自动安装安全（和其他）更新。如果你计划使用它，你要有一些方法监控你的系统，例如，同时安装 apt-listchanges软件包并设置它给你发送有关更新的邮件。

  1. 先安装程序

    apt-get install unattended-upgrades

  2. 然后执行

    dpkg-reconfigure unattended-upgrades //选择yes

  3. 修改配置文件

    vim  /etc/apt/apt.conf.d/50unattended-upgrades  //配置
    vim /etc/apt/apt.conf.d/20auto-upgrades  //更新周期
